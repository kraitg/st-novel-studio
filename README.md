# Novel Studio 小说工坊

一个 **SillyTavern / TauriTavern 第三方扩展** 的调试骨架。依赖node.js,是用前请先安装依赖,
当前版本只做一件事：

- 在**扩展栏（魔杖菜单 `#extensionsMenu`）**注入一个「**开始创意**」入口
- 点击后打开一个**全屏面板 GUI**
- 面板顶部**状态栏**：显示连接状态、当前 AI 来源+模型、当前预设，并随酒馆设置变化实时刷新
- **全局关键字**：一栏引导词，作为所有 AI 生成内容的引导。以系统角色注入每次生成，全局持久化保存
- **写新小说模块**：输入背景设定与章节大纲，逐章生成正文；手动/自动两种模式；每 20 章自动总结存档并注入后续生成
- **续写小说 · 上传分析**：点击上传 txt，分段并发分析建立角色档案与剧情梗概（可折叠展示）；随后可按大纲续写或自然续写
- 面板内含运行环境自检与调试按钮

## 续写小说 · 上传分析

> 卡片标题栏有「清空所有内容，开始新项目」按钮：弹窗确认后清空续写会话、上传分析结果与全部写新小说项目（保留全局关键字与分析参数），回到初始状态。

面板「续写小说 · 上传分析」区域，用于把已有的整本小说导入并分析：

1. **上传小说**：点击「点击上传 txt 文档」按钮选择 `.txt` 文件（UTF-8 自动读取，支持十几万字），右侧显示文件名与字数。
2. **参数**：
   - **每段字数**（默认 6000）：切段粒度。
   - **并发**（默认 2）：同时分析的段数。过高可能触发 API 限流。
3. **开始分析** → 进度条显示提取/聚合进度，可随时「中止」。

### 高效分析原理（分段并发 + 分层聚合）

- **分段**：全文按"每段字数"切分，尽量在段落换行或中文句末标点处断开，避免切碎句子。
- **并发提取**：用限并发执行器（默认 2 并发）对各段并行调用 API，各自提取"角色 + 剧情要点"。这是十几万字仍能较快完成的关键。
- **分层聚合**：段数很多时先按每 15 段一批合并去重，再总合并，避免单次 prompt 过长；最终产出**全书角色档案 + 剧情梗概**。
- **结果展示**：分析完成后**不创建小说项目、不切换界面**；结果保存在续写模块自身状态，以两行可折叠展开的形式（角色档案 / 剧情梗概）展示在上传分析卡片内，点击标题行即可展开/收起。
- 分析生成同样用 `generateRaw`（优先 TavernHelper，降级原生），不写入聊天记录；中止用逐 `genId` 停止或 `stopAllGeneration`。

### 开始续写

分析完成后，续写区出现在分析结果下方，续写数据（角色档案、剧情梗概、原文末尾、续写章节等）**持久化**存于 `extensionSettings.novel_studio.continuation`，刷新不丢失。两种模式：

1. **按大纲续写**（章 - 节层次）：「添加章」建大章（含章标题+章总览），每章「添加节」建节并填节大纲，「保存大纲」保存。
   - **手动续写（逐节确认）**：生成下一未完成节后停下。
   - **自动续写（全部）**：依次生成所有未完成节，可中止。已有内容时提供「全部重新生成」（弹窗确认后清空已生成正文，按当前章-节大纲从头重写）。
   - 生成每节前预读所属章的章总览；每写完一整章生成整章表格总结。
2. **自然续写**：只填「续写章数」，AI 自主逐章续写（每章一节），前一节结尾作为下一节上下文，直到达到设定章数。已有内容时可「全部重新生成」。

**续写上下文**：角色档案 + 剧情梗概 + 最新整章总结 +（大纲模式）本章总览 +（首节）原文末尾若干字（`contextTailChars`，默认 2500）+ 最近节结尾。生成走 `generateRaw`，不写入聊天，可中止。

## 写新小说模块

在面板「写新小说」区域：

1. **新建小说**：可建多本，下拉切换。数据存于 `extensionSettings.novel_studio.projects`。
2. **背景设定**：世界观、人物、写作要求等。
3. **主要人物角色 / 道具设定**：逐行添加（人物与道具混合），每节生成都会作为固定【人物与道具设定】块注入 prompt。
4. **大纲（章 - 节层次）**：
   - 「添加章」新增一大章，每章含**章标题**与**章总览**（本章整体走向）。
   - 每章下「添加节」新增节，每节填**节大纲**。
   - 点「保存背景/设定/大纲」保存整个章-节结构。
5. **生成（以"节"为单位）**：
   - **手动模式（逐节确认）**：每次生成"下一个未完成节"，生成后停下。
   - **自动模式（全部生成）**：依次生成所有未完成节，可中止。
   - 也可对单节点「生成本节 / 重新生成」。
   - **生成每节前会预读所属章的「章总览」** 作为上下文，再写本节，保证与本章走向一致。
6. **整章表格总结**：**每写完一整章**（该章所有节都生成完）后，自动调用 AI 对该章所有节生成**表格总结**（人物表 + 剧情表）并存档；后续生成把**最新一条总结**作为长期记忆注入，强化连贯。
7. **章节操作**：每个已生成节可「查看全文」「复制」；「一键导出(txt)」按章-节结构导出全书。

### 生成调用说明

- 使用 `generateRaw` 语义（**优先** `TavernHelper.generateRaw`，**降级**原生 `generateRaw`），**不携带**角色卡/世界书/聊天历史，只用：背景设定 + 人物设定 + 最新整章总结 + 本章总览 + 最近 2 节正文 + 本节大纲。
- 采样参数仍来自当前 API/预设。生成不会写入聊天记录。
- 中止：优先 `TavernHelper.stopGenerationById`，降级 `stopAllGeneration` / 原生 `stopGeneration`。
- 旧版平铺章节数据会**自动迁移**为章-节（每个旧章 → 1 大章含 1 节，已生成正文保留）。

## 全局关键字（引导词）

面板内「全局关键字」卡片可输入一段引导词，作为**所有 AI 生成内容**的引导：

- **作用域**：全局（所有聊天/角色卡共享），存于扩展全局设置 `extensionSettings.novel_studio`，通过 `saveSettingsDebounced()` 持久化。
- **注入方式**：通过 `setExtensionPrompt()`，以 `IN_CHAT` + `SYSTEM` 角色、depth=4 注入，对 **Chat Completion 与 Text Completion 均生效**。
- **时机**：每次生成前监听 `GENERATION_STARTED` 事件动态刷新注入；关键字为空或关闭"启用注入"时自动清除注入。
- 常量（`extension_prompt_types`/`extension_prompt_roles`）采用字面量，避免相对 `import` 路径依赖目录深度，兼容 TauriTavern。

**不包含任何写小说业务功能**，用于验证扩展加载、按钮注入、GUI 打开/关闭与依赖检测是否正常。

## 安装（放入扩展目录）

将整个 `st-novel-studio` 目录放到扩展目录，二选一：

- 所有用户：`SillyTavern/public/scripts/extensions/third-party/st-novel-studio`
- 指定用户：`SillyTavern/data/<user>/extensions/st-novel-studio`

TauriTavern 同理，放到其前端的第三方扩展目录（`public/scripts/extensions/third-party/`）。

放好后刷新页面，在「扩展 / 管理扩展」里确认已启用。

> 调试期建议用**符号链接**代替复制，改代码后刷新页面即可生效：
>
> ```powershell
> # 以管理员 PowerShell 运行, 路径按实际修改
> New-Item -ItemType SymbolicLink `
>   -Path "C:\path\to\SillyTavern\public\scripts\extensions\third-party\st-novel-studio" `
>   -Target "D:\opencode\st-novel-studio"
> ```

## 与酒馆助手 (JS-Slash-Runner / TavernHelper) 的关系

本框架把酒馆助手当作**可选增强**：

- 如果安装并启用了酒馆助手，`window.TavernHelper` 会被检测到（面板自检里显示"已就绪"）。
- 如果没有，扩展仍能正常加载并打开面板（降级模式）。

后续接入写小说业务时，再通过 `window.TavernHelper` 与 `SillyTavern.getContext()` 调用其 API。

## 调试方法

1. 打开浏览器/WebView 开发者工具（F12）→ Console。
2. 过滤关键字 `NovelStudio`，可看到初始化、注入、点击、开关面板等日志。
3. 点击扩展栏「开始创意」或右下角悬浮羽毛按钮打开面板。
4. 面板内：
   - **重新自检**：刷新 ST context / TavernHelper 检测结果。
   - **打印一条调试日志**：验证事件绑定与日志链路。
5. `Esc` 或点击遮罩空白处可关闭面板。

## 说明 / 注意

- `manifest.json` 的 `optional` 声明了 `third-party/JS-Slash-Runner`，使本扩展在酒馆助手之后加载，但不强制依赖。
- 按钮注入采用**轮询重试**（`#extensionsMenu` 可能异步渲染），失败时回退到右下角悬浮按钮。
- 面板样式尽量复用 ST 主题 CSS 变量（如 `--SmartThemeBodyColor`），自动适配明/暗主题。
